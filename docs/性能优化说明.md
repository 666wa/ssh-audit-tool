# SSH审计工具 - 性能优化说明

## 优化内容

### 1. 支持CSV格式（重要）

**问题**：Excel文件读取速度慢，特别是大文件（40万+行）

**解决方案**：
- 支持CSV格式输入和输出
- CSV读取速度比Excel快5-10倍
- 自动识别文件格式（.xlsx, .xls, .csv）

**使用方法**：
```bash
# 将Excel转换为CSV（推荐）
# 在Excel中：文件 -> 另存为 -> CSV UTF-8

# 使用CSV文件运行
python run_audit.py -o 操作命令表.csv -r 报备表.csv --output 结果.csv
```

### 2. 建立索引加速查询

**问题**：44万条报备记录，每次匹配都要遍历全部记录

**解决方案**：
- 为每个账号建立索引
- 匹配时只查询该账号相关的记录
- 查询速度提升10-100倍（取决于数据分布）

**原理**：
```
之前：遍历44万条记录
现在：只遍历该账号的记录（通常几百到几千条）
```

### 3. 数据清洗

**优化**：
- 自动移除空行
- 跳过无效数据
- 减少内存占用

## 推荐配置

### 大数据量场景（10万+记录）

1. **转换为CSV格式**
   ```bash
   # 在Excel中转换
   文件 -> 另存为 -> CSV UTF-8 (逗号分隔)(*.csv)
   ```

2. **修改配置文件**
   ```python
   # ssh_audit_tool/config.py
   DEFAULT_OPERATION_FILE = r"D:\...\操作命令表.csv"
   DEFAULT_REPORT_FILE = r"D:\...\报备表.csv"
   DEFAULT_OUTPUT_DIR = r"D:\Desktop"
   ```

3. **运行工具**
   ```bash
   python run_audit.py
   ```

### 输出格式选择

- **CSV格式**（推荐）：
  - 文件小
  - 读写快
  - 可用Excel打开

- **Excel格式**：
  - 格式美观
  - 支持公式
  - 文件较大

## 内存优化

### 当前实现
- 一次性加载所有数据到内存
- 适用于：报备表 < 100万行

### 如需处理更大数据
可以考虑：
1. 分块处理（chunk processing）
2. 使用数据库（SQLite）
3. 使用Dask等大数据工具

## 注意事项

1. **CSV编码**：
   - 使用 UTF-8 with BOM 编码
   - Excel另存为时选择"CSV UTF-8"

2. **CSV分隔符**：
   - 使用逗号分隔
   - 不要使用分号或制表符

3. **特殊字符**：
   - CSV中的逗号会被正确处理
   - 换行符会被转义

4. **Excel兼容性**：
   - CSV文件可以直接用Excel打开
   - 双击打开或在Excel中导入

## 故障排查

### CSV文件乱码
**解决方案**：
- 使用"CSV UTF-8"格式保存
- 或在Excel中：数据 -> 从文本/CSV -> 选择UTF-8编码

### 内存不足
**解决方案**：
- 关闭其他程序
- 使用64位Python
- 考虑分块处理

### 处理速度仍然慢
**检查**：
- 是否使用了CSV格式？
- 报备表是否有重复数据？
- 是否有大量无效记录？
