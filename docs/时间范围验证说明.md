# 时间范围验证功能说明

## 概述

SSH绕行核查工具现在支持**时间范围验证**功能。当报备表包含"生效时间"和"失效时间"列时，工具会自动验证操作时间是否在报备的有效期内。

## 功能特性

### 1. 自动检测
- 工具启动时自动检测报备表是否包含时间列
- 如果包含，自动启用时间范围验证
- 如果不包含，跳过时间验证（保持原有逻辑）

### 2. 验证逻辑

#### 情况1：操作时间在有效期内
```
生效时间: 2025/01/01 00:00
失效时间: 2025/12/31 23:59
操作时间: 2025/06/15 10:00

结果: 已报备 ✓
```

#### 情况2：操作时间已过期
```
生效时间: 2025/01/01 00:00
失效时间: 2025/01/15 23:59
操作时间: 2025/01/20 10:00

结果: 已报备但已过期 ✗
```

#### 情况3：操作时间早于生效时间
```
生效时间: 2025/01/01 00:00
失效时间: 2025/12/31 23:59
操作时间: 2024/12/15 10:00

结果: 未报备 ✗
```

#### 情况4：无失效时间（永久有效）
```
生效时间: 2025/01/01 00:00
失效时间: （空）
操作时间: 2026/06/15 10:00

结果: 已报备 ✓
```

#### 情况5：无生效时间
```
生效时间: （空）
失效时间: 2025/12/31 23:59
操作时间: 2025/06/15 10:00

结果: 已报备 ✓
```

## 报备表格式要求

### 必需列
- 序号
- 访问账号
- 本端主机IP
- 对端主机IP

### 可选列（时间验证）
- **生效时间**：报备开始生效的时间
- **失效时间**：报备失效的时间

### 时间格式支持
工具自动识别以下时间格式：
- `2025/01/01 00:00`
- `2025-01-01 00:00:00`
- `2025-01-01 00:00`
- `2025/01/01 00:00:00`
- `2025/01/01`
- `2025-01-01`

## 操作表格式要求

### 原始模板
- 必需列：**操作时间**

### 4A模板
- 必需列：**op_time**（操作时间）

## 结果说明

### 核查结果类型
1. **已报备**：三元组匹配且在有效期内
2. **已报备但已过期**：三元组匹配但已过期
3. **未报备**：三元组不匹配或早于生效时间
4. **异常：非有效ssh命令**：SSH命令无效
5. **异常：无法获取有效本端IP**：无法提取本端IP

### 命中序号
- 已报备：显示报备表序号
- 已报备但已过期：显示报备表序号
- 其他情况：空

## 使用示例

### 示例1：报备表包含时间列
```csv
序号,访问账号,本端主机IP,对端主机IP,生效时间,失效时间
1,user1,10.1.1.1,10.2.2.1,2025/01/01 00:00,2025/12/31 23:59
2,user2,10.1.1.2,10.2.2.2,2025/01/01 00:00,2025/01/15 23:59
```

**操作记录**：
```
操作时间: 2025/01/20 10:00
SSH命令: ssh user2@10.2.2.2
资源IP: 10.1.1.2
```

**核查结果**：
```
对比结果: 已报备但已过期
命中序号: 2
```

### 示例2：报备表不包含时间列
```csv
序号,访问账号,本端主机IP,对端主机IP
1,user1,10.1.1.1,10.2.2.1
```

**操作记录**：
```
操作时间: 2025/01/20 10:00
SSH命令: ssh user1@10.2.2.1
资源IP: 10.1.1.1
```

**核查结果**：
```
对比结果: 已报备
命中序号: 1
```
（不验证时间，保持原有逻辑）

## 日志输出

### 启用时间验证
```
[2026-01-22 10:00:00] INFO - 报备表包含时间范围列，将验证操作时间是否在生效期内
```

### 未启用时间验证
```
[2026-01-22 10:00:00] INFO - 报备表不包含时间范围列，跳过时间验证
```

## 注意事项

1. **时间列可选**：如果报备表不包含时间列，工具自动跳过时间验证
2. **时间格式灵活**：支持多种常见时间格式，自动识别
3. **空值处理**：
   - 生效时间为空：不限制开始时间
   - 失效时间为空：永久有效
   - 两者都为空：等同于不验证时间
4. **向后兼容**：不影响现有报备表的使用

## 常见问题

**Q: 如果报备表只有生效时间或失效时间，会怎样？**
A: 工具仍然会启用时间验证，只验证存在的时间边界。

**Q: 时间格式不对会怎样？**
A: 工具会尝试多种格式解析，如果都失败，则跳过该记录的时间验证。

**Q: 已过期的报备算已报备吗？**
A: 算，但会标记为"已报备但已过期"，便于识别需要更新的报备记录。

**Q: 会影响性能吗？**
A: 影响很小，时间解析只在匹配成功后进行，且使用了高效的日期解析方法。
