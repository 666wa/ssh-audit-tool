### 2025-12-29 - 增强SSH命令验证逻辑

### 增强的验证规则

为了更准确地识别无效SSH命令，我们在原有验证逻辑基础上增加了以下4项严格检查：

#### 1. 用户名非法字符检测
**规则**：
- 用户名不能以点（`.`）开头
- 用户名不能包含非ASCII字符（中文、特殊符号等）
- 用户名只能包含：字母、数字、下划线、连字符、点号（但不能在开头）

**拒绝示例**：
```bash
ssh .cmhop@10.246.26.186        # 用户名以点开头
ssh bonc鈥斺_cs@localhost        # 用户名包含中文字符
```

#### 2. Shell重定向符号检测
**规则**：
- 命令中不能包含shell重定向符号：`>`, `>>`, `<`, `|`

**拒绝示例**：
```bash
ssh 10.246.26.28 cat ~/.ssh/id_rsa.pub >>~/.ssh/authorized_keys
# 包含 >> 重定向符号
```

**原因**：SSH命令本身不应该包含shell重定向，这些通常是命令执行错误或误操作。

#### 3. 文件扩展名检测
**规则**：
- SSH命令中目标前不能出现带扩展名的文件名
- 检测的扩展名包括：`.sh`, `.py`, `.zip`, `.tar`, `.gz`, `.txt`, `.log`, `.conf`, `.cfg`, `.json`, `.xml`, `.yaml`, `.yml`

**拒绝示例**：
```bash
ssh jdkchange.sh 10.232.143.75                    # 脚本名在IP前
ssh python36.zip cmhop@10.232.217.81:...          # 文件名在用户@IP前
```

**原因**：这些通常是用户混淆了SSH和SCP命令，或者是误操作。

#### 4. -l参数格式检测
**规则**：
- `-l` 参数后应该只跟用户名，不能拼接其他内容（如密码、密钥）
- 检测模式：`-l username:...`（冒号后有内容）

**拒绝示例**：
```bash
ssh 10.252.65.198 -l dacp:rDaCooqsNCFFsYYCdsYqsqwNCYCCCwaqqpxpikSrBiQQ=G|T
# -l 参数后拼接了密码或密钥内容
```

**原因**：正确的格式应该是 `ssh -l username ip`，密码应该在连接后输入，不应该在命令行中。

## 验证流程

### 完整的验证检查顺序

```
1. 命令非空检查
   ↓
2. 包含"ssh"关键字检查
   ↓
3. IPv6格式检查（拒绝）
   ↓
4. Shell重定向符号检查（拒绝）  ← 新增
   ↓
5. 文件扩展名检查（拒绝）        ← 新增
   ↓
6. -l参数格式检查（拒绝）        ← 新增
   ↓
7. 提取目标IP（必须成功）
   ↓
8. 提取用户名（如果有）
   ↓
9. 用户名非法字符检查（拒绝）    ← 新增
   ↓
✅ 验证通过
```

## 影响范围

### 对现有数据的影响

增强验证后，以下类型的命令会被标记为"异常：非有效ssh命令"：

1. **包含shell操作的命令**
   - 之前：可能被识别为"未报备"
   - 现在：标记为"异常：非有效ssh命令"

2. **文件操作混用的命令**
   - 之前：可能被识别为"未报备"
   - 现在：标记为"异常：非有效ssh命令"

3. **参数格式错误的命令**
   - 之前：可能被识别为"未报备"
   - 现在：标记为"异常：非有效ssh命令"

4. **用户名包含非法字符的命令**
   - 之前：可能被识别为"未报备"
   - 现在：标记为"异常：非有效ssh命令"

### 优势

1. **更准确的分类**：将真正无效的命令与未报备的有效命令区分开
2. **减少误判**：避免将格式错误的命令当作有效命令进行匹配
3. **提高数据质量**：帮助识别数据源中的问题命令
4. **便于审计**：清晰区分"未报备"和"命令格式错误"两种情况

## 使用建议

### 运行测试
```bash
# 运行验证测试
python test_invalid_commands.py
```

### 查看结果
运行核查工具后，检查"对比结果"列：
- **已报备** - 命令有效且已报备
- **未报备** - 命令有效但未报备
- **异常：非有效ssh命令** - 命令格式错误（包括新增的4类检查）
- **异常：无法获取有效本端IP** - 无法提取本端IP

### 数据清理建议
如果发现大量"异常：非有效ssh命令"，建议：
1. 检查数据源质量
2. 确认是否有批量数据录入错误
3. 与数据提供方确认这些命令的真实性
