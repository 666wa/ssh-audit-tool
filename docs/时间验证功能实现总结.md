# 时间范围验证功能实现总结

## 实现概述

已成功为SSH绕行核查工具添加**时间范围验证**功能，支持验证操作时间是否在报备的有效期内。

## 实现范围

### ✅ 已实现的脚本

1. **run_audit.py**（原始模板）
   - 调用 `ssh_audit_tool/main.py`
   - 自动读取操作表中的"操作时间"列
   - 传递给核查器进行时间验证

2. **run_audit_4a.py**（4A模板）
   - 调用 `ssh_audit_tool/main_4a.py`
   - 自动读取操作表中的"op_time"列
   - 传递给核查器进行时间验证

### 核心逻辑

两个脚本都使用相同的核查器（`ssh_audit_tool/auditor.py`），因此时间验证逻辑完全一致。

## 修改的文件

### 1. 配置文件
- `ssh_audit_tool/config.py`
  - 添加 `OPTIONAL_REPORT_TIME_COLUMNS`（生效时间、失效时间）
  - 添加 `RESULT_EXPIRED`（已报备但已过期）

- `ssh_audit_tool/config_4a.py`
  - 添加相同的时间列配置
  - 添加相同的过期结果标记

### 2. 核心模块
- `ssh_audit_tool/auditor.py`
  - 添加 `check_time_columns()` 方法：检测报备表是否包含时间列
  - 修改 `check_match()` 方法：添加 `operation_time` 参数，验证时间范围
  - 修改 `audit_record()` 方法：传递操作时间
  - 修改 `audit_record_4a()` 方法：传递操作时间
  - 修改 `audit_record_with_target_ip()` 方法：传递操作时间

### 3. 主程序
- `ssh_audit_tool/main.py`
  - 读取操作表的"操作时间"列
  - 传递给 `auditor.audit_record()`

- `ssh_audit_tool/main_4a.py`
  - 读取操作表的"op_time"列
  - 传递给 `auditor.audit_record()` 和 `auditor.audit_record_with_target_ip()`

### 4. 文件处理
- `ssh_audit_tool/file_handler_4a.py`
  - 修改 `read_report_table()` 方法：读取所有列（包括时间列）
  - 移除了 `usecols` 限制，确保时间列被读取

## 功能特性

### 1. 自动检测
```python
def check_time_columns(self) -> None:
    if '生效时间' in self.report_df.columns and '失效时间' in self.report_df.columns:
        self.has_time_columns = True
        logger.info("报备表包含时间范围列，将验证操作时间是否在生效期内")
    else:
        self.has_time_columns = False
        logger.info("报备表不包含时间范围列，跳过时间验证")
```

### 2. 时间格式支持
支持多种常见时间格式：
- `2025/01/01 00:00`
- `2025-01-01 00:00:00`
- `2025-01-01 00:00`
- `2025/01/01 00:00:00`
- `2025/01/01`
- `2025-01-01`

### 3. 验证逻辑
```python
# 验证时间范围
if effective_dt and op_datetime < effective_dt:
    # 操作时间早于生效时间，不匹配
    continue

if expiry_dt and op_datetime > expiry_dt:
    # 操作时间晚于失效时间，返回特殊标记（已过期）
    return f"EXPIRED:{serial_number}"
```

### 4. 结果类型
- **已报备**：三元组匹配且在有效期内
- **已报备但已过期**：三元组匹配但已过期
- **未报备**：三元组不匹配或早于生效时间
- **异常：非有效ssh命令**：SSH命令无效
- **异常：无法获取有效本端IP**：无法提取本端IP

## 使用方法

### 方法1：报备表包含时间列
```csv
序号,访问账号,本端主机IP,对端主机IP,生效时间,失效时间
1,user1,10.1.1.1,10.2.2.1,2025/01/01 00:00,2025/12/31 23:59
```

运行：
```bash
python run_audit.py
# 或
python run_audit_4a.py
```

工具会自动：
1. 检测到时间列
2. 启用时间验证
3. 输出结果包含"已报备但已过期"状态

### 方法2：报备表不包含时间列
```csv
序号,访问账号,本端主机IP,对端主机IP
1,user1,10.1.1.1,10.2.2.1
```

运行：
```bash
python run_audit.py
# 或
python run_audit_4a.py
```

工具会自动：
1. 检测到无时间列
2. 跳过时间验证
3. 使用原有逻辑（向后兼容）
