# 4A绕行核实模板使用说明

## 概述

本工具现在支持两种操作记录模板：
1. **原始模板**：月-使用ssh命令连接其它设备-SDC数据源
2. **4A模板**：4A绕行核实_大数据分公司

两种模板使用相同的核查逻辑，只是列名不同。

## 4A模板列名映射

| 4A模板列名（英文） | 4A模板列名（中文） | 原始模板列名 | 用途说明 |
|-------------------|-------------------|-------------|---------|
| op_fort_content | 操作内容 | 操作内容 | SSH命令 |
| res_ip | 资源ip | 资源IP | **本端IP** |
| res_name | 资源名称 | 资源名称 | 资源名称（本端IP备用） |
| main_acct_name | 主账号名称 | 主账号名称 | 主账号 |
| sub_acct_name | 从账号名称 | 从账号名称 | 从账号（优先使用） |

### 与原始模板的区别

| 项目 | 原始模板 | 4A模板 | 说明 |
|------|---------|--------|------|
| 本端IP来源 | 资源IP → 资源名称 | res_ip → res_name | **逻辑相同** |
| 对端IP来源 | SSH命令提取 | SSH命令提取 | **逻辑相同** |
| 列名类型 | 中文 | 英文 | 仅列名不同 |
| 核查逻辑 | 三元组匹配 | 三元组匹配 | **完全一致** |

## 文件结构

4A模板CSV文件的表头：
- 第1行：英文列名（id, audit_desc, resp_org, ..., op_fort_content, res_ip, ...）

注意：实际文件只有一行英文表头，工具会直接读取。

## 使用方法

### 方法1：使用快速运行脚本（推荐）

1. 修改配置文件 `ssh_audit_tool/config_4a.py`：
```python
DEFAULT_OPERATION_FILE = r"D:\path\to\4A绕行核实_大数据分公司.csv"
DEFAULT_REPORT_FILE = r"D:\path\to\主机互访报备.csv"
DEFAULT_OUTPUT_DIR = r"D:\path\to\output"
```

2. 运行脚本：
```bash
python run_audit_4a.py
```

### 方法2：使用命令行参数

```bash
python -m ssh_audit_tool.main_4a -o 4A绕行核实表.csv -r 报备表.csv
```

可选参数：
- `--output 结果.csv`：指定输出文件路径
- `--no-timestamp`：输出文件名不添加时间戳
- `--log-level INFO`：设置日志级别（DEBUG/INFO/WARNING/ERROR）

## 核查逻辑

### 基本逻辑（与原始模板一致）

4A模板和原始模板使用**基本相同**的核查逻辑，只是列名不同（英文 vs 中文）。

1. **本端IP获取**：res_ip（资源IP） → res_name（资源名称） → 异常
2. **对端IP获取**：SSH命令提取 → 异常
3. **账号选择**：SSH命令中的用户 → sub_acct_name（从账号） → main_acct_name（主账号）
4. **SSH命令验证**：22种无效命令检测
5. **三元组匹配**：(账号, 本端IP, 对端IP)
6. **时间范围验证**：自动验证操作时间是否在报备有效期内（如果报备表包含生效时间和失效时间列）

### 特殊逻辑（4A模板专有）

根据**审计说明（audit_desc）**字段判断对端IP的来源：

#### 情况1：过堡垒机跳板绕行
当audit_desc包含"**过堡垒机操作存在跳板绕行**"时：
- **对端IP来源**：直接使用server_ip（目的地址）
- **原因**：执行命令的IP与单点登录资源IP不同，SSH命令中的IP不是真实对端IP
- **示例**：登录到A机器后，又跳转到B机器执行命令，此时应该用B机器的IP（server_ip）

#### 情况2：正常绕行
当audit_desc为其他内容时：
- **对端IP来源**：从SSH命令提取
- **原因**：SSH命令中的IP就是真实对端IP

## 输出结果

在原始CSV基础上添加两列：
- **对比结果**：已报备 / 未报备 / 已报备但已过期 / 异常：非有效ssh命令 / 异常：无法获取有效本端IP
- **命中序号**：匹配的报备表序号（已报备时显示）

### 时间范围验证
如果报备表包含"生效时间"和"失效时间"列，工具会自动验证操作时间是否在有效期内：
- **已报备**：三元组匹配且在有效期内
- **已报备但已过期**：三元组匹配但已过期
- **未报备**：三元组不匹配或早于生效时间

详见：[时间范围验证说明](时间范围验证说明.md)

## 示例

### 示例1：正常情况
```
audit_desc: 主机存在绕行行为，操作命令使用跳板命令，且无SMC报备
op_fort_content: ssh cmhop@10.252.98.210
res_ip: 10.252.98.210  (资源IP，作为本端IP)
server_ip: 10.252.98.210  (目的地址，不使用)
sub_acct_name: oper_yw
```

**核查过程**：
1. 本端IP：10.252.98.210（从res_ip获取）
2. SSH命令：ssh cmhop@10.252.98.210（有效）
3. 账号：cmhop（从SSH命令提取）
4. 对端IP：10.252.98.210（**从SSH命令提取**）
5. 三元组：(cmhop, 10.252.98.210, 10.252.98.210)

### 示例2：过堡垒机跳板绕行
```
audit_desc: 过堡垒机操作存在跳板绕行，执行命令的IP与单点登录资源IP不同...
op_fort_content: ssh cmhop@10.1.1.1
res_ip: 10.2.2.2  (登录的资源IP，作为本端IP)
server_ip: 10.3.3.3  (实际执行命令的IP，作为对端IP)
sub_acct_name: oper_yw
```

**核查过程**：
1. 本端IP：10.2.2.2（从res_ip获取）
2. 审计说明：包含"过堡垒机操作存在跳板绕行" → **使用特殊逻辑**
3. 账号：cmhop（从SSH命令提取）
4. 对端IP：10.3.3.3（**直接使用server_ip，忽略SSH命令中的IP**）
5. 三元组：(cmhop, 10.2.2.2, 10.3.3.3)

### 输出
```
对比结果: 已报备 / 未报备
命中序号: 1234（如果已报备）
```

## 注意事项

1. **CSV格式**：4A模板必须是CSV格式，有一行英文表头
2. **编码**：自动识别UTF-8/GBK/GB2312编码
3. **账号分隔符**：报备表中支持 中文逗号(，)、顿号(、)、斜杠(/)
4. **不支持英文逗号**：避免与CSV格式冲突
5. **性能**：CSV格式比Excel快5-10倍
6. **报备表编码**：如果报备表是GBK编码，工具会自动识别

## 常见问题

**Q: 为什么要分开两个版本？**
A: 两种模板的列名不同，分开处理更清晰，避免混淆。

**Q: 核查逻辑有区别吗？**
A: 完全相同，只是读取的列名不同。

**Q: 可以用同一个报备表吗？**
A: 可以，报备表格式相同。

**Q: 输出格式一样吗？**
A: 一样，都是添加"对比结果"和"命中序号"两列。
