# SSH核查工具 - 核心逻辑说明

## 核查流程

### 1. 数据读取
- 读取操作命令表（支持Excel/CSV）
- 读取报备表（支持Excel/CSV）
- 建立账号索引（加速查询）

### 2. 本端IP获取
**优先级**：
1. 从"资源IP"列获取
2. 如果无效（localhost/127.0.0.1/空），从"资源名称"列提取
3. 都无法获取 → **异常：无法获取有效本端IP**

### 3. SSH命令验证
**有效条件**：
- 包含"ssh"关键字
- 能提取出有效IPv4地址
- 不是错误格式（如 `ssh ip@user`）
- localhost必须精确匹配（不匹配localhost8等）
- 不包含shell重定向符号（`>`, `>>`, `<`, `|`）
- 不在目标前包含文件名（如 `.sh`, `.zip`, `.py` 等）
- -l参数格式正确（不拼接密码或密钥）
- 用户名不包含非法字符（不以点开头，不包含非ASCII字符）

**无效** → **异常：非有效ssh命令**

### 4. 账号提取
**优先级**：
1. SSH命令中指定的用户（如 `ssh user@ip` 中的user）
2. 从账号名称（如果存在）
3. 主账号名称

### 5. 对端IP提取
- 从SSH命令中提取目标IP
- 如果是 `localhost` 或 `127.0.0.1` → 替换为本端IP

### 6. 三元组匹配
**匹配条件**（必须全部满足）：
- **账号**：在报备表"访问账号"中（支持顿号分隔的多账号）
- **本端IP**：匹配报备表"本端主机IP"（支持IP段、CIDR、连续范围）
- **对端IP**：匹配报备表"对端主机IP"（支持IP段、CIDR、连续范围）

**结果**：
- 全部匹配 → **已报备**（返回序号）
- 无法匹配 → **未报备**

## 关键特性

### IP格式支持
- 精确IP：`192.168.1.1`
- IP段：`192.168.1.1-192.168.1.20`
- 连续范围：`10.1.1.1-10.1.1.254-10.1.2.1-10.1.2.254`
- CIDR：`192.168.1.0/24`
- 混合格式：`192.168.1.1、192.168.1.10-192.168.1.20`（顿号分隔）

### SSH命令格式
**支持**：
- `ssh user@ip`
- `ssh -l user ip`
- `ssh user ip`
- `ssh ip`（使用默认账号）
- `ssh localhost`（转换为本端IP）

**拒绝**：
- `ssh ip@user`（格式错误）
- `ssh localhost8`（无效主机名）
- 不包含ssh关键字
- 无法提取有效IP

### 性能优化
- **账号索引**：只查询相关账号的记录
- **CSV格式**：读取速度提升5-10倍
- **44万报备记录**：处理时间约10分钟

## 输出结果

| 结果 | 说明 |
|------|------|
| 已报备 | 三元组完全匹配，显示报备序号 |
| 未报备 | 三元组无法匹配 |
| 异常：非有效ssh命令 | SSH命令格式错误或无法解析 |
| 异常：无法获取有效本端IP | 资源IP和资源名称都无法提取IP |

## 核心代码模块

- **parser.py**：SSH命令解析、格式验证
- **auditor.py**：三元组匹配、索引查询
- **ip_matcher.py**：IP格式匹配（段、CIDR、范围）
- **file_handler.py**：文件读写（Excel/CSV）
- **utils.py**：本端IP提取、工具函数
- **main.py**：主流程控制

## 配置文件

**config.py** 中的三个全路径配置：
```python
DEFAULT_OPERATION_FILE  # 操作命令表路径
DEFAULT_REPORT_FILE     # 报备表路径
DEFAULT_OUTPUT_DIR      # 输出目录路径
```

## 使用建议

1. **大数据量**：转换为CSV格式
2. **默认运行**：`python run_audit.py`
3. **自定义**：修改 `config.py` 中的路径配置
