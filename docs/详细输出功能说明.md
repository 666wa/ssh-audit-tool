# 详细输出功能说明

## 概述

SSH绕行核查工具现在支持**详细输出**功能，在输出结果中新增"详细说明"列，清晰展示每个字段的匹配情况。

## 输出列

### 原有列
1. **对比结果**：已报备 / 未报备 / 已报备但已过期 / 异常
2. **命中序号**：匹配的报备表序号

### 新增列
3. **详细说明**：详细的匹配信息（新增）

## 详细说明格式

### 1. 已报备
```
已报备：命中序号1，访问时间段2025/01/01 00:00~2025/12/31 23:59，访问账号admin，源端IP10.1.1.1，对端IP10.2.2.1
```

**包含信息：**
- 命中序号：报备表中的序号
- 访问时间段：生效时间~失效时间（如果报备表包含时间列）
- 访问账号：匹配的账号
- 源端IP：匹配的本端IP
- 对端IP：匹配的对端IP

### 2. 已报备但已过期
```
已报备但已过期：命中序号2，访问时间段2025/01/01 00:00~2025/01/15 23:59，访问账号user1，源端IP10.1.1.2，对端IP10.2.2.2
```

**包含信息：**
- 命中序号：报备表中的序号
- 访问时间段：生效时间~失效时间（显示已过期的时间段）
- 访问账号：匹配的账号
- 源端IP：匹配的本端IP
- 对端IP：匹配的对端IP

### 3. 未报备
```
未报备，访问账号admin已命中，源端IP10.9.9.9未命中，对端IP10.2.2.1未命中
```

**包含信息：**
- 访问账号：是否命中（已命中/未命中）
- 源端IP：是否命中（已命中/未命中）
- 对端IP：是否命中（已命中/未命中）

## 使用示例

### 示例1：完全匹配
**操作记录：**
```
操作时间: 2025/06/15 10:00
SSH命令: ssh admin@10.2.2.1
资源IP: 10.1.1.1
```

**报备记录：**
```
序号: 1
访问账号: admin
本端主机IP: 10.1.1.1
对端主机IP: 10.2.2.1
生效时间: 2025/01/01 00:00
失效时间: 2025/12/31 23:59
```

**输出结果：**
```
对比结果: 已报备
命中序号: 1
详细说明: 已报备：命中序号1，访问时间段2025/01/01 00:00~2025/12/31 23:59，访问账号admin，源端IP10.1.1.1，对端IP10.2.2.1
```

### 示例2：已过期
**操作记录：**
```
操作时间: 2025/01/20 10:00
SSH命令: ssh user1@10.2.2.2
资源IP: 10.1.1.2
```

**报备记录：**
```
序号: 2
访问账号: user1
本端主机IP: 10.1.1.2
对端主机IP: 10.2.2.2
生效时间: 2025/01/01 00:00
失效时间: 2025/01/15 23:59  ← 已过期
```

**输出结果：**
```
对比结果: 已报备但已过期
命中序号: 2
详细说明: 已报备但已过期：命中序号2，访问时间段2025/01/01 00:00~2025/01/15 23:59，访问账号user1，源端IP10.1.1.2，对端IP10.2.2.2
```

### 示例3：账号不匹配
**操作记录：**
```
操作时间: 2025/06/15 10:00
SSH命令: ssh unknown@10.2.2.1
资源IP: 10.1.1.1
```

**输出结果：**
```
对比结果: 未报备
命中序号: 
详细说明: 未报备，访问账号unknown未命中，源端IP10.1.1.1未命中，对端IP10.2.2.1未命中
```

### 示例4：源端IP不匹配
**操作记录：**
```
操作时间: 2025/06/15 10:00
SSH命令: ssh admin@10.2.2.1
资源IP: 10.9.9.9  ← 不在报备表中
```

**输出结果：**
```
对比结果: 未报备
命中序号: 
详细说明: 未报备，访问账号admin已命中，源端IP10.9.9.9未命中，对端IP10.2.2.1未命中
```

### 示例5：对端IP不匹配
**操作记录：**
```
操作时间: 2025/06/15 10:00
SSH命令: ssh admin@10.9.9.9  ← 不在报备表中
资源IP: 10.1.1.1
```

**输出结果：**
```
对比结果: 未报备
命中序号: 
详细说明: 未报备，访问账号admin已命中，源端IP10.1.1.1未命中，对端IP10.9.9.9未命中
```

### 示例6：永久有效
**操作记录：**
```
操作时间: 2026/06/15 10:00
SSH命令: ssh user2@10.2.2.3
资源IP: 10.1.1.3
```

**报备记录：**
```
序号: 3
访问账号: user2
本端主机IP: 10.1.1.3
对端主机IP: 10.2.2.3
生效时间: 2025/01/01 00:00
失效时间: （空）  ← 永久有效
```

**输出结果：**
```
对比结果: 已报备
命中序号: 3
详细说明: 已报备：命中序号3，访问时间段2025/01/01 00:00~永久，访问账号user2，源端IP10.1.1.3，对端IP10.2.2.3
```

## 优势

### 1. 快速定位问题
通过详细说明，可以快速了解：
- 哪个字段匹配了
- 哪个字段没匹配
- 为什么未报备

### 2. 便于审计
- 已报备：显示完整的匹配信息，便于审计追溯
- 未报备：显示具体哪个字段不匹配，便于补充报备

### 3. 时间范围可见
- 显示报备的有效时间段
- 识别已过期的报备
- 显示永久有效的报备

## 常见问题

**Q: 详细说明会影响性能吗？**
A: 不会，详细说明是在匹配过程中同步生成的，不增加额外开销。

**Q: 可以关闭详细说明吗？**
A: 详细说明列始终输出，如果不需要可以在Excel中隐藏该列。

**Q: 详细说明支持中文吗？**
A: 是的，详细说明使用中文，便于阅读和理解。

**Q: 如果报备表不包含时间列，详细说明会显示什么？**
A: 会显示"无限制"，表示不限制时间范围。
