# SSH命令核查工具

## 简介

SSH命令核查工具用于自动化验证实际执行的SSH操作命令是否已在报备表中完成报备。系统通过比对操作命令表和报备表中的关键信息（访问账号、本端IP、对端IP），自动判断每条SSH操作是否已完成报备，并生成包含核查结果的输出文件。

## 功能特点

- ✅ 自动解析SSH命令，提取访问账号和对端IP
- ✅ 支持多种SSH命令格式（`ssh user@ip`、`ssh -l user ip`、`ssh user ip`）
- ✅ 支持复杂IP格式匹配（特定IP、IP段、连续IP段、子网掩码）
- ✅ 支持报备表中多个IP混合格式（以顿号分隔）
- ✅ 支持连续用"-"分隔的多个IP范围（如`10.1.1.1-10.1.1.254-10.1.2.1-10.1.2.254`）
- ✅ 自动修复异常IP格式（多个点、中文句号、特殊字符）
- ✅ 生成详细的核查结果和统计信息
- ✅ 显示命中的报备记录序号
- ✅ 完整的日志记录，便于问题排查

## 使用方法

### 方式一：使用默认文件（快速开始）

默认文件路径已配置在 `ssh_audit_tool/config.py` 中（全路径）：

```python
DEFAULT_OPERATION_FILE = r"D:\Desktop\gaoyf工作交接\ssh绕行脚本\ssh-niu\月-使用ssh命令连接其它设备-SDC数据源(每日分析)_2025-12-09_09-58-30-661_602.xlsx"
DEFAULT_REPORT_FILE = r"D:\Desktop\gaoyf工作交接\ssh绕行脚本\ssh-niu\主机互访报备 (12月9日).xlsx"
DEFAULT_OUTPUT_DIR = r"D:\Desktop\gaoyf工作交接\ssh绕行脚本\ssh-niu"
```

直接运行：

```bash
python run_audit.py
```

**修改默认路径**：编辑 `ssh_audit_tool/config.py` 文件，修改这三个配置项为你的实际路径。

### 方式二：指定文件路径

```bash
python run_audit.py -o <操作命令表.xlsx> -r <报备表.xlsx>
```

示例：

```bash
python run_audit.py -o 操作命令表.xlsx -r 报备表.xlsx
```

### 方式三：自定义输出路径

```bash
python run_audit.py -o 操作命令表.xlsx -r 报备表.xlsx --output 自定义结果.xlsx
```

### 方式四：添加时间戳到输出文件名

默认情况下，输出文件名不包含时间戳。如果需要添加时间戳：

```bash
# 不添加时间戳（默认）
python run_audit.py -o 操作命令表.xlsx -r 报备表.xlsx

# 输出：结果-操作命令表.xlsx
```

如果不使用`--no-timestamp`选项，可以通过修改代码默认行为来添加时间戳。

### 方式五：使用模块方式运行

```bash
python -m ssh_audit_tool.main -o <操作命令表.xlsx> -r <报备表.xlsx>
```

## 命令行参数

| 参数 | 简写 | 说明 | 必需 | 默认值 |
|------|------|------|------|--------|
| `--operation` | `-o` | 操作命令表文件路径 | 是 | - |
| `--report` | `-r` | 报备表文件路径 | 是 | - |
| `--output` | - | 自定义输出文件路径 | 否 | 自动生成 |
| `--no-timestamp` | - | 输出文件名不添加时间戳 | 否 | False |
| `--log-level` | - | 日志级别（DEBUG/INFO/WARNING/ERROR） | 否 | INFO |

**输出文件命名规则：**
- 默认：`结果-<原文件名>.xlsx`（在原文件同目录）
- 使用`--output`：自定义路径和文件名
- 不使用`--no-timestamp`时：`结果-<原文件名>.xlsx`
- 使用时间戳（需修改代码）：`结果-<原文件名>_YYYYMMDD_HHMMSS.xlsx`

## 输入文件格式

### 操作命令表（F1）

必需包含以下列：
- **操作时间**: SSH操作的时间
- **资源IP**: 本端IP（发起SSH连接的源IP）
- **操作内容**: SSH命令内容（如`ssh cmhop@10.187.245.250`）
- **主账号名称**: 主账号名称

**支持的异常IP格式自动修复：**
- 多个连续的点：`10..102..80..146` → `10.102.80.146`
- 中文句号：`10.246。163.30` → `10.246.163.30`
- 全角点号：`10．246．163．30` → `10.246.163.30`
- 中文字符：`10中文.246测试.163.30` → `10.246.163.30`
- 特殊字符：`10.246銆銆.163.30` → `10.246.163.30`
- 混合异常：`10..246。163銆.30中文` → `10.246.163.30`
- **自动移除所有中文字符和非ASCII特殊字符**

### 报备表（F2）

必需包含以下列：
- **访问账号**: SSH使用的账号名
- **本端主机IP**: 本端IP地址（支持特定IP、IP段、子网掩码、多IP混合）
- **对端主机IP**: 对端IP地址（支持特定IP、IP段、子网掩码、多IP混合）

**支持的IP格式：**
- 特定IP：`192.168.1.1`
- IP段：`192.168.1.1-192.168.1.20`
- 连续IP段：`10.230.72.1-10.230.72.254-10.230.73.1-10.230.73.254`（多个IP段用"-"连续分隔，两两配对）
- 子网掩码：`192.168.1.0/24`
- 多IP混合：`192.168.1.1、192.168.1.10-192.168.1.20、10.0.0.0/8`（以顿号分隔）

## 输出结果

### 输出文件

程序会在操作命令表所在目录生成结果文件，文件名格式为：

```
结果-<原文件名>.xlsx
```

例如：`结果-月-使用ssh命令连接其它设备-SDC数据源(每日分析)_2025-12-09_09-58-30-661_602.xlsx`

### 核查结果说明

输出文件会在原操作命令表基础上添加两列：

#### 对比结果列

| 结果 | 说明 |
|------|------|
| **已报备** | 访问账号、本端IP、对端IP三者均在报备表中匹配 |
| **未报备** | 三者无法在报备表中完全匹配 |
| **异常：非有效ssh命令** | 操作内容不是有效的SSH命令（不包含ssh关键字或无法提取有效IP），不作为参考依据 |

#### 命中序号列

显示匹配到的报备表中"序号"列的值：
- 当"对比结果"为"已报备"时，显示匹配的报备记录的序号（如：1、2、3等）
- 当"对比结果"为"未报备"或"-"时，该列为空

**注意**：命中序号是报备表中"序号"列的实际值，不是Excel的行号。

### 统计信息

程序运行完成后会显示统计信息，例如：

```
[2025-12-25 17:03:07] INFO - 核查完成，共处理 1198 条记录
[2025-12-25 17:03:07] INFO -   已报备: 450 条 (37.6%)
[2025-12-25 17:03:07] INFO -   未报备: 717 条 (59.8%)
[2025-12-25 17:03:07] INFO -   异常：非有效ssh命令: 31 条 (2.6%)
```

### 输出示例

输出文件包含原始数据的所有列，并在最后添加两列：

| 操作时间 | 资源IP | 操作内容 | 主账号名称 | ... | 对比结果 | 命中序号 |
|---------|--------|---------|-----------|-----|---------|---------|
| 2025-11-26 14:58:44 | 10.246.163.30 | ssh cmhop@10.187.245.250 | chaizhidan1 | ... | 已报备 | 3 |
| 2025-11-26 14:04:21 | 10.103.137.94 | ssh cmhop@10.103.137.93 | dwxutenglong | ... | 未报备 | |
| 2025-11-26 13:20:15 | 10.209.231.50 | ssh omm@localhost | user123 | ... | 已报备 | 15 |

**说明**：命中序号显示的是报备表中"序号"列的值，方便快速定位到具体的报备记录。

## 日志文件

程序运行时会生成日志文件`ssh_audit.log`，记录详细的处理过程和错误信息，便于问题排查。

## 常见问题

### 1. 提示"缺少必需的列"

**原因**：Excel文件缺少必需的列名。

**解决方案**：
- 检查操作命令表是否包含：操作时间、资源IP、操作内容、主账号名称
- 检查报备表是否包含：访问账号、本端主机IP、对端主机IP

### 2. 提示"文件不存在"

**原因**：指定的文件路径不正确。

**解决方案**：
- 检查文件路径是否正确
- 使用绝对路径或相对路径
- 确保文件名和扩展名正确

### 3. 提示"无法写入文件（权限不足）"

**原因**：输出文件可能已被Excel打开，或没有写入权限。

**解决方案**：
- 关闭已打开的结果文件
- 检查输出目录的写入权限
- 尝试以管理员身份运行

### 4. 所有记录都显示"未报备"

**原因**：可能是数据格式不匹配或报备表数据有问题。

**解决方案**：
- 检查报备表中的IP格式是否正确
- 检查访问账号是否匹配
- 查看日志文件`ssh_audit.log`获取详细信息

### 5. 大量记录显示"异常：非有效ssh命令"

**原因**：操作内容不包含ssh关键字或无法提取出有效的IP地址

**解决方案**：
- 检查操作内容格式是否正确
- 确认是否为有效的SSH命令
- "异常：非有效ssh命令"的记录不作为参考依据，可以忽略
- 注意：系统目前不支持IPv6地址
- 注意：系统不会自动修复异常IP格式（如多个点、中文字符等）
