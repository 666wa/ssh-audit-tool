# 违规使用模板核查逻辑详解

## 问题说明

违规使用模板中的操作内容（`op_fort_content`）大部分**不是标准的SSH命令**，而是各种Linux命令，如：
- `ll`
- `ip addr`
- `clear`
- `tail -f nohup.out`
- `sh deploy.sh`

那么，核查逻辑是如何工作的呢？

## 核查逻辑流程

### 1. 字段提取

从违规使用CSV中提取关键字段：

```python
# 从CSV中提取字段
operation_content = row.get('op_fort_content')    # 操作内容：ll, clear等
server_ip = row.get('server_ip')                  # 服务器IP：10.102.44.103（本端IP）
dst_ip = row.get('dst_ip')                        # 目的地址：可能为空
client_ip = row.get('client_ip')                  # 客户端IP：10.250.104.83
account = row.get('sub_acct_name')                # 从账号：webapp, midapp等
```

**实际数据示例**：

**类型1：自然人使用程序账号无工单授权记录**
```
操作内容: ll
服务器IP: 10.102.44.103
目的地址: （空）
客户端IP: 10.250.104.83
从账号: webapp
```

**类型2：过堡垒机操作存在跳板绕行**
```
操作内容: top
服务器IP: 10.246.123.51
目的地址: 10.246.123.187
客户端IP: 10.250.104.83
从账号: dwliuzhiwei
```

### 2. 确定对端IP

根据`dst_ip`是否有值来确定对端IP：

```python
# 确定对端IP
if dst_ip and dst_ip != 'nan' and dst_ip.strip():
    target_ip = dst_ip              # 优先使用目的地址
else:
    target_ip = client_ip           # 如果目的地址为空，使用客户端地址
```

**逻辑说明**：
- **有dst_ip**：说明是跳板绕行场景，对端IP = dst_ip
- **无dst_ip**：说明是直接访问场景，对端IP = client_ip

### 3. SSH命令构造

因为操作内容不是SSH命令，所以需要**构造一个SSH命令格式**来进行核查：

```python
# 构造SSH命令格式
constructed_cmd = f"ssh {account}@{target_ip}"
```

**构造示例**：

**类型1示例**：
```
原始操作内容: ll
对端IP: 10.250.104.83（来自client_ip）
构造的SSH命令: ssh webapp@10.250.104.83
```

**类型2示例**：
```
原始操作内容: top
对端IP: 10.246.123.187（来自dst_ip）
构造的SSH命令: ssh dwliuzhiwei@10.246.123.187
```

### 4. 核查三元组

使用构造的SSH命令进行核查，核查的三元组是：

```python
# 核查三元组（只核查IP对，不考虑账号）
source_ip = server_ip           # 本端IP：服务器地址
target_ip = dst_ip or client_ip # 对端IP：目的地址或客户端地址
account = sub_acct_name         # 账号（用于构造命令，但核查时IP更重要）
```

**类型1核查三元组**：
```
account = "webapp"
source_ip = "10.102.44.103"     # 从server_ip提取（本端IP）
target_ip = "10.250.104.83"     # 从client_ip提取（对端IP）
```

**类型2核查三元组**：
```
account = "dwliuzhiwei"
source_ip = "10.246.123.51"     # 从server_ip提取（本端IP）
target_ip = "10.246.123.187"    # 从dst_ip提取（对端IP）
```

### 4. 与报备表对比

在报备表中查找匹配的记录：

```python
# 报备表的匹配条件（只核查IP对）
报备表.本端主机IP 包含 source_ip
报备表.对端主机IP 包含 target_ip
# 注意：账号匹配是次要的，主要看IP对是否匹配
```

## 关键代码解析

### 代码片段1：字段提取
```python
# 提取字段（使用英文列名）
operation_content = str(row.get('op_fort_content', ''))
client_ip = str(row.get('client_ip', ''))      # 源IP
server_ip = str(row.get('server_ip', ''))      # 目标IP
account = str(row.get('sub_acct_name', ''))    # 从账号名称
```

### 代码片段2：SSH命令构造
```python
# 如果操作内容不是SSH命令格式，尝试构造
if not operation_content.strip().startswith('ssh'):
    # 构造SSH命令格式：ssh account@server_ip
    if account and account != 'nan' and server_ip and server_ip != 'nan':
        constructed_cmd = f"ssh {account}@{server_ip}"
    else:
        # 无法构造，标记为无效命令
        results.append(RESULT_INVALID_COMMAND)
        continue
else:
    # 已经是SSH命令，直接使用
    constructed_cmd = operation_content
```

### 代码片段3：核查调用
```python
# 使用client_ip作为源IP
source_ip = client_ip

# 核查单条记录
result, matched_row, detail = auditor.audit_record(
    constructed_cmd,  # 构造的SSH命令
    source_ip,        # 客户端IP作为源IP
    account           # 从账号名称
)
```

## 为什么这样设计？

### 1. 业务场景理解

违规使用模板记录的是**两种不同的场景**：

**场景1：自然人使用程序账号（dst_ip为空）**
```
用户 → 堡垒机(client_ip) → 目标服务器(server_ip) → 执行命令(op_fort_content)
```
这种情况下，用户从client_ip通过堡垒机连接到server_ip执行命令。

**场景2：跳板绕行（dst_ip有值）**
```
用户 → 堡垒机(client_ip) → 跳板机(server_ip) → 目标服务器(dst_ip) → 执行命令
```
这种情况下，用户先连接到跳板机server_ip，然后从跳板机连接到dst_ip执行命令。

### 2. 核查本质

核查的本质是验证：**IP对之间的连接权限**

- **场景1**：验证从server_ip到client_ip的连接权限
- **场景2**：验证从server_ip到dst_ip的连接权限

所以需要根据dst_ip是否有值来确定对端IP：
```python
target_ip = dst_ip if dst_ip else client_ip
```

### 3. 与报备表的对应关系

报备表记录的是**IP对之间的连接权限**：
```
本端主机IP: 10.102.44.103
对端主机IP: 10.250.104.83
```

这表示：允许从10.102.44.103连接到10.250.104.83

**注意**：账号匹配是次要的，主要看IP对是否匹配。

## 常见问题

### Q1: 为什么要构造SSH命令？
**A**: 因为核查的本质是验证IP对之间的连接权限，而不是验证具体执行的命令。操作内容（ll、clear等）只是在连接建立后执行的命令。

### Q2: dst_ip为空时怎么办？
**A**: 如果dst_ip为空，说明是直接访问场景，使用client_ip作为对端IP。如果dst_ip有值，说明是跳板绕行场景，使用dst_ip作为对端IP。

### Q3: 本端IP和对端IP分别是什么？
**A**: 
- **本端IP**：始终是server_ip（服务器地址）
- **对端IP**：优先使用dst_ip（目的地址），如果为空则使用client_ip（客户端地址）

### Q4: 账号的作用是什么？
**A**: 账号主要用于构造SSH命令格式，但核查时**IP对的匹配更重要**。用户要求"只核查IP对，不考虑账号"。

### Q5: 为什么大部分记录都是"未报备"？
**A**: 可能的原因：
1. 这些IP对确实没有报备
2. 报备表中的IP范围不包含这些IP对
3. 报备已过期
4. IP对的方向不匹配（本端和对端颠倒）

### Q6: 两种场景的区别是什么？
**A**: 
- **自然人使用程序账号**（dst_ip为空）：用户从堡垒机直接访问目标服务器
- **跳板绕行**（dst_ip有值）：用户从堡垒机先到跳板机，再从跳板机访问目标服务器

## 总结

违规使用模板的核查逻辑：

1. **提取字段**：从CSV中提取server_ip、dst_ip、client_ip、账号
2. **确定对端IP**：优先使用dst_ip，如果为空则使用client_ip
3. **构造SSH命令**：`ssh account@target_ip`
4. **核查IP对**：(本端=server_ip, 对端=target_ip)
5. **与报备表对比**：查找匹配的IP对报备记录
6. **返回结果**：已报备/未报备/已报备但已过期/异常

**核心理念**：
- 核查的是**IP对之间的连接权限**，而不是具体的命令权限
- **只核查IP对，账号是次要的**
- 根据dst_ip是否有值来区分两种场景：
  - **dst_ip为空**：直接访问场景，对端IP = client_ip
  - **dst_ip有值**：跳板绕行场景，对端IP = dst_ip

**数据流向**：
```
场景1（dst_ip为空）：
  堡垒机(client_ip) ← → 目标服务器(server_ip)
  核查：server_ip ← → client_ip

场景2（dst_ip有值）：
  堡垒机(client_ip) → 跳板机(server_ip) → 目标服务器(dst_ip)
  核查：server_ip ← → dst_ip
```
