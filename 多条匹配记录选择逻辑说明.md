# 多条匹配记录选择逻辑说明

## 问题背景

当一个操作记录（账号+源端IP+对端IP）能够匹配到多条报备记录时，需要确定选择哪一条记录进行判断。

## 原始逻辑的问题

**原始逻辑**：按报备表顺序遍历，找到第一条IP匹配的记录就立即返回。

**存在的问题**：
1. 如果第一条记录已过期，但后面有有效记录，会错误地返回"已报备但已过期"
2. 没有优先选择最合适的记录
3. 可能导致误判

**示例问题**：
```
报备记录：
- 序号1001: 2026/01/01 ~ 2026/01/31 (已过期)
- 序号1002: 2026/01/15 ~ 2026/02/28 (有效)
- 序号1003: 2026/02/01 ~ 2026/03/31 (有效)

文件日期: 2026-02-10
原始逻辑结果: 返回序号1001 "已报备但已过期" ❌
期望结果: 返回有效的记录 ✅
```

## 改进后的逻辑

### 选择策略

**优先级1：有效期内的记录**
- 文件日期在生效时间和失效时间之间
- 如果有多条有效记录，选择**生效时间最晚的**（最新的报备）

**优先级2：已过期的记录**
- 文件日期晚于失效时间
- 如果有多条过期记录，选择**失效时间最晚的**（最近过期的）

**优先级3：未生效的记录**
- 文件日期早于生效时间
- 这些记录被跳过，返回"未报备"

### 实现逻辑

1. **收集所有匹配记录**：遍历该账号的所有记录，收集IP匹配的记录
2. **分类记录**：
   - `valid_matches`：有效期内的记录
   - `expired_matches`：已过期的记录
   - 未生效的记录被跳过
3. **选择最佳记录**：
   - 优先从`valid_matches`中选择生效时间最晚的
   - 如果没有有效记录，从`expired_matches`中选择失效时间最晚的

### 核心方法

```python
def _select_best_valid_match(self, valid_matches: list) -> dict:
    """从有效期内的记录中选择最佳匹配（生效时间最晚的）"""

def _select_best_expired_match(self, expired_matches: list) -> dict:
    """从已过期的记录中选择最佳匹配（失效时间最晚的）"""
```

## 改进效果

### 测试结果对比

| 文件日期 | 原始逻辑 | 改进逻辑 | 说明 |
|---------|---------|---------|------|
| 2026-01-10 | 序号1001 ✅ | 序号1001 ✅ | 只有1001有效 |
| 2026-02-10 | 序号1001 (过期) ❌ | 序号1003 ✅ | 选择最新的有效记录 |
| 2026-03-10 | 序号1001 (过期) ❌ | 序号1003 ✅ | 选择唯一的有效记录 |

### 优势

1. **准确性提升**：优先返回有效的报备记录，避免误判
2. **智能选择**：在多条记录中选择最相关的（最新的有效记录或最近过期的记录）
3. **逻辑清晰**：明确的优先级策略，便于理解和维护

## 使用场景

这个改进特别适用于以下场景：

1. **报备记录更新**：新的报备记录覆盖旧的记录
2. **时间范围重叠**：多个报备记录的时间范围有重叠
3. **长期运维**：需要确保选择最新、最相关的报备记录

## 总结

通过改进多条匹配记录的选择逻辑，系统现在能够：

- ✅ 优先返回有效期内的记录
- ✅ 在多条有效记录中选择最新的
- ✅ 在多条过期记录中选择最相关的
- ✅ 避免因记录顺序导致的误判

这确保了审计结果的准确性和可靠性。
