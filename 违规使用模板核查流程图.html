SSH审计工具 - 账号违规使用模板核查逻辑可视化

1 从CSV文件提取字段
账号 (sub_acct_name): webapp
源IP (client_ip): 10.250.104.83
目标IP (server_ip): 10.102.44.103
操作内容 (op_fort_content): ll
↓
2 判断操作内容是否为SSH命令
检查 op_fort_content 是否以 "ssh" 开头
↓
❌ 否 (大部分情况)
操作内容是普通命令：
ll
clear
tail -f nohup.out
sh deploy.sh
→ 需要构造SSH命令
ssh webapp@10.102.44.103
✅ 是 (少数情况)
操作内容已经是SSH命令：
ssh root@10.1.1.1
ssh admin@10.1.1.5
→ 直接使用该命令
ssh root@10.1.1.1
↓
3 提取核查三元组
访问账号: webapp
源端IP: 10.250.104.83 (来自 client_ip)
对端IP: 10.102.44.103 (来自 server_ip)
↓
4 在报备表中查找匹配记录
匹配条件1: 报备表.访问账号 = "webapp"
匹配条件2: 报备表.本端主机IP 包含 "10.250.104.83"
匹配条件3: 报备表.对端主机IP 包含 "10.102.44.103"
匹配条件4: 文件日期在生效时间和失效时间之间
↓
5 返回核查结果
✅ 已报备 - 找到匹配的有效报备记录
❌ 未报备 - 未找到匹配的报备记录
⏰ 已报备但已过期 - 找到匹配但报备已过期
⚠️ 异常 - 无法构造有效的SSH命令
💡 业务场景理解
违规使用模板记录的是通过堡垒机或跳板机的操作行为：
用户 → 堡垒机(client_ip) → 目标服务器(server_ip) → 执行命令(ll, clear等)
虽然用户执行的是 ll、clear 等普通命令，
但这些命令是在通过SSH连接到目标服务器后执行的。

核查的本质：验证用户是否有权限从client_ip通过SSH连接到server_ip
📋 实际示例
输入数据：
sub_acct_name: webapp
client_ip: 10.250.104.83
server_ip: 10.102.44.103
op_fort_content: ll
处理过程：
1. 检测到 "ll" 不是SSH命令
2. 构造SSH命令: ssh webapp@10.102.44.103
3. 提取三元组: (webapp, 10.250.104.83, 10.102.44.103)
4. 在报备表中查找匹配记录
核查结果：
未报备
详细说明：未报备，访问账号webapp已命中，源端IP10.250.104.83未命中，对端IP10.102.44.103未命中

💻 关键代码片段
# 1. 提取字段
operation_content = row.get('op_fort_content')
client_ip = row.get('client_ip')
server_ip = row.get('server_ip')
account = row.get('sub_acct_name')

# 2. 构造SSH命令
if not operation_content.strip().startswith('ssh'):
    # 不是SSH命令，构造SSH命令格式
    constructed_cmd = f"ssh {account}@{server_ip}"
else:
    # 已经是SSH命令，直接使用
    constructed_cmd = operation_content

# 3. 设置源IP
source_ip = client_ip

# 4. 核查
result, matched_row, detail = auditor.audit_record(
    constructed_cmd,  # 构造的SSH命令
    source_ip,        # 客户端IP作为源IP
    account           # 从账号名称
)
🎯 核心要点总结
1. 为什么要构造SSH命令？
    因为核查的本质是验证SSH连接权限，而不是验证具体执行的命令。

2. 账号从哪里来？
    来自CSV的 sub_acct_name 字段（从账号名称）。

3. 源IP和目标IP从哪里来？
    源IP：来自 client_ip 字段（客户端地址）
    目标IP：来自 server_ip 字段（服务器地址）

4. 为什么大部分记录都是"未报备"？
    可能的原因：
    • 这些操作确实没有报备
    • 报备表中的IP范围不包含这些IP
    • 报备已过期
    • 账号名称不匹配
