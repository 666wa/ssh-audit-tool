# 账号违规使用模板使用说明

## 概述

`run_audit_violation.py` 是专门为处理"账号违规使用_大数据分公司-策略反馈"格式CSV文件而设计的审计工具。

## 模板特点

### CSV文件格式
- **列名**：使用英文列名（id, audit_desc, client_ip, server_ip等）
- **第二行**：包含中文列名说明（会被自动跳过）
- **编码**：UTF-8-BOM格式

### 关键字段映射

| 原始列名 | 说明 | 用途 |
|---------|------|------|
| `sub_acct_name` | 从账号名称 | 用于匹配报备表的访问账号 |
| `client_ip` | 客户端地址 | 作为本端IP（源IP） |
| `server_ip` | 服务器地址 | 作为对端IP（目标IP） |
| `op_fort_content` | 操作内容 | SSH命令或操作描述 |
| `op_time` | 操作时间 | 操作发生的时间 |

## 使用方法

### 方式1：使用默认配置
```bash
python run_audit_violation.py
```

默认配置：
- 操作文件：`2026-02-03\账号违规使用_大数据分公司-策略反馈-20260203_095931.csv`
- 报备表：`2026-01-22\主机互访报备-20260122.csv`
- 输出目录：`2026-02-03`

### 方式2：指定文件路径
```bash
python run_audit_violation.py -o 违规使用.csv -r 报备表.csv
```

### 方式3：指定输出文件
```bash
python run_audit_violation.py -o 违规使用.csv -r 报备表.csv --output 结果.csv
```

### 方式4：不添加时间戳
```bash
python run_audit_violation.py --no-timestamp
```

## 命令行参数

| 参数 | 说明 | 必需 |
|------|------|------|
| `-o, --operation` | 账号违规使用CSV文件路径 | 否（有默认值） |
| `-r, --report` | 报备表文件路径 | 否（有默认值） |
| `--output` | 输出文件路径 | 否（自动生成） |
| `--no-timestamp` | 输出文件名不添加时间戳 | 否 |
| `--log-level` | 日志级别（DEBUG/INFO/WARNING/ERROR） | 否（默认INFO） |

## 特殊处理逻辑

### 1. SSH命令构造
如果`op_fort_content`不是标准SSH命令格式，工具会自动构造：
```
原始操作内容: "ll"
构造命令: "ssh account@server_ip"
```

### 2. IP地址处理
- **源IP**：直接使用`client_ip`字段
- **目标IP**：直接使用`server_ip`字段
- 不需要从资源名称中提取IP

### 3. 时间验证
- 自动从文件名提取日期（如：20260203）
- 使用提取的日期进行时间范围验证
- 支持报备表的生效时间和失效时间验证

### 4. 性能优化
- 默认使用高性能审计器（FastAuditor）
- 支持大规模数据处理（45万+报备记录）
- 自动缓存时间解析和IP匹配结果

## 输出结果

### 输出文件格式
- **文件名**：`结果-原文件名_时间戳.csv`
- **编码**：UTF-8-BOM
- **格式**：CSV

### 新增列

| 列名 | 说明 |
|------|------|
| `对比结果` | 核查结果（已报备/未报备/已报备但已过期/异常） |
| `命中序号` | 匹配的报备表序号 |
| `详细说明` | 详细的匹配信息 |

### 核查结果类型

1. **已报备**：找到匹配的有效报备记录
2. **未报备**：未找到匹配的报备记录
3. **已报备但已过期**：找到匹配但报备已过期
4. **异常：非有效ssh命令**：操作内容无法解析为有效SSH命令
5. **异常：无法获取有效本端IP**：源IP无效或缺失

## 与其他模板的区别

### 普通模板（run_audit.py）
- 列名：中文（操作时间、资源IP、操作内容等）
- 源IP：从"资源IP"或"资源名称"提取
- 账号：优先使用"从账号名称"，否则使用"主账号名称"

### 4A模板（run_audit_4a.py）
- 列名：中文
- 源IP：从"源地址"字段
- 目标IP：从"目的地址"字段或SSH命令提取
- 特殊处理：支持堡垒机跳板场景

### 违规使用模板（run_audit_violation.py）
- 列名：英文
- 源IP：直接使用`client_ip`
- 目标IP：直接使用`server_ip`
- 特殊处理：自动构造SSH命令格式

## 注意事项

1. **文件编码**：确保CSV文件使用UTF-8-BOM编码
2. **列名完整性**：必须包含所有必需列（sub_acct_name, client_ip, server_ip, op_fort_content, op_time）
3. **报备表格式**：报备表必须包含标准列名（访问账号、本端主机IP、对端主机IP）
4. **性能考虑**：处理大量数据时建议使用高性能模式（默认已启用）
5. **日志文件**：运行日志保存在`ssh_audit.log`

## 故障排除

### 问题1：找不到文件
```
错误: 操作文件不存在
解决: 检查文件路径是否正确，使用绝对路径或相对路径
```

### 问题2：缺少必需列
```
错误: 缺少必需的列: ['sub_acct_name']
解决: 检查CSV文件是否包含所有必需列，确认列名拼写正确
```

### 问题3：编码问题
```
错误: UnicodeDecodeError
解决: 确保CSV文件使用UTF-8-BOM编码保存
```

### 问题4：性能慢
```
解决: 
1. 确认USE_FAST_AUDITOR=True（默认已启用）
2. 减少报备表记录数量
3. 使用更强大的硬件
```
